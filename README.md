# 🎉 생일 축하 이벤트 웹사이트

친구의 22번째 생일을 위한 특별한 빈티지 필름 카메라 컨셉 웹사이트입니다.

## 📱 사용 환경
- **권장 기기**: iPhone 13 이상
- **브라우저**: Safari (모바일)
- **화면 방향**: 세로 모드

## 🎵 음악 파일 준비하기

웹사이트를 완성하려면 두 개의 음악 파일이 필요합니다:

### 1. BGM (추억 사진 슬라이드용)
- **파일명**: `bgm.mp3`
- **위치**: `assets/audio/bgm.mp3`
- **용도**: 추억 사진이 나올 때 재생되는 배경음악
- **추천**: 친구와의 추억이 담긴 노래, 잔잔한 음악

### 2. 생일축하 노래
- **파일명**: `birthday.mp3`
- **위치**: `assets/audio/birthday.mp3`
- **용도**: 마지막 케이크 섹션에서 재생
- **추천**: 생일축하 노래 또는 축하 메시지가 담긴 음성

### 음악 파일 추가 방법
```bash
# assets/audio 폴더에 두 파일을 복사하세요
assets/
  └── audio/
      ├── bgm.mp3
      └── birthday.mp3
```

## 📸 추억 사진 커스터마이징

현재 `script.js` 파일의 `memories` 배열에서 사진과 캡션을 수정할 수 있습니다:

```javascript
const memories = [
    { img: 'assets/stickers/파일명.jpg', caption: '원하는 메시지' },
    // ... 추가 사진들
];
```

### 사진 추가/변경 방법
1. `assets/stickers/` 폴더에 사진 파일 저장
2. `script.js`의 `memories` 배열에서 파일명과 캡션 수정
3. 순서대로 슬라이드에 표시됩니다

## 🚀 실행 방법

### 1. 로컬에서 테스트
```bash
# Python이 설치되어 있다면
python -m http.server 8000

# 또는 Node.js를 사용한다면
npx http-server
```

그 다음 브라우저에서 `http://localhost:8000` 접속

### 2. 모바일에서 테스트
- 같은 Wi-Fi에 연결된 상태에서
- 컴퓨터의 IP 주소로 접속 (예: `http://192.168.0.10:8000`)
- 또는 GitHub Pages, Netlify 등에 배포하여 접속

## 🎬 진행 순서

1. **오프닝**: 필름이 스크롤되며 "HAPPY BIRTHDAY" 메시지 표시
2. **추억 사진**: 사진을 클릭/탭하면 위로 넘어가며 다음 사진 표시
3. **카메라**: 빈티지 카메라로 친구 사진 촬영 및 저장
4. **케이크**: 생일 케이크와 함께 축하 메시지 표시

## ⚙️ 기능 설명

### 터치/클릭 인터랙션
- 오프닝: "TAP TO START" 클릭으로 시작
- 추억 사진: 사진을 탭하면 다음 슬라이드로 넘어감
- 카메라: 촬영 버튼(📷)으로 사진 찍기
- 사진 저장: SAVE PHOTO 버튼으로 다운로드

### 음악 자동 재생
- **추억 사진 섹션**: BGM 자동 재생
- **카메라 섹션**: 음악 없음
- **케이크 섹션**: 생일축하 노래 자동 재생

> 참고: 모바일 브라우저는 자동 재생을 차단할 수 있습니다.
> 첫 터치 시 음악이 재생되도록 설정되어 있습니다.

### 카메라 권한
- 최초 카메라 섹션 진입 시 카메라 권한 요청
- 사용자 얼굴 카메라(전면 카메라) 사용
- 촬영한 사진은 폴라로이드 스타일로 표시

## 🎨 디자인 컨셉

- **빈티지 필름 카메라**: 필름 스트립 프레임 디자인
- **색상 테마**: 핑크 계열 (#ffa5c6, #ff6b9d) + 다크 배경
- **폰트**: Courier New (빈티지 타이포그래피)
- **애니메이션**: 부드러운 슬라이드, 케이크 성장 애니메이션

## 🐛 문제 해결

### 카메라가 작동하지 않을 때
- HTTPS 또는 localhost에서만 카메라 접근 가능
- 브라우저 설정에서 카메라 권한 확인
- iPhone의 경우 Safari 사용 권장

### 음악이 재생되지 않을 때
- 음악 파일이 `assets/audio/` 폴더에 있는지 확인
- 파일명이 `bgm.mp3`, `birthday.mp3`인지 확인
- 모바일에서는 첫 터치 후 재생됨 (자동재생 정책)

### 사진이 표시되지 않을 때
- 파일 경로 확인: `assets/stickers/파일명.jpg`
- 파일 확장자 확인 (.jpg, .png 등)
- `script.js`의 memories 배열에서 경로 재확인
